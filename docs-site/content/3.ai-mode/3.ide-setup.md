---
title: IDE Setup
description: Configure your IDE to use ZVault as an MCP tool provider.
---

`zvault setup` generates the correct MCP configuration for your IDE so the ZVault MCP server is automatically available.

## Cursor

```bash
zvault setup cursor
```

Creates:
- `.cursor/mcp.json` — MCP server configuration
- `.cursor/rules/zvault.mdc` — Cursor rules for safe secret handling

The rules file tells Cursor to never output real secret values and to use `zvault://` references.

## Kiro

```bash
zvault setup kiro
```

Creates:
- `.kiro/settings/mcp.json` — MCP server configuration with auto-approve for read-only tools
- `.kiro/steering/zvault.md` — Steering file with ZVault usage instructions

Read-only tools (`zvault_list_secrets`, `zvault_describe_secret`, `zvault_check_env`, `zvault_generate_env_template`, `zvault_vault_status`) are auto-approved. Write tools require confirmation.

## Continue

```bash
zvault setup continue
```

Creates:
- `.continue/config.json` — MCP server configuration

## Generic (llms.txt)

```bash
zvault setup generic
```

Creates:
- `llms.txt` — A plain-text file that tells any AI tool how to work with ZVault

Use this for tools that don't support MCP but can read project context files.

## Smart Merge

If a config file already exists, `zvault setup` merges the ZVault MCP server entry into it without overwriting your existing configuration. Other MCP servers you've configured are preserved.

## Environment Requirements

All IDE setups require these environment variables to be set:

```bash
export VAULT_ADDR="http://127.0.0.1:8200"
export VAULT_TOKEN="hvs.your-token-here"
```

The generated configs reference `${VAULT_TOKEN}` so your token is resolved from the environment at runtime, not hardcoded in config files.

## Verifying Setup

After running setup, ask your AI assistant:

> "What secrets does this project use?"

It should use the `zvault_list_secrets` tool and return a list of secret paths without any actual values.
