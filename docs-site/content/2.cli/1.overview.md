---
title: CLI Overview
description: Complete reference for the ZVault command-line interface.
---

The `zvault` CLI is a standalone binary that communicates with the ZVault server over HTTP. It has no internal dependencies — it talks exclusively via the REST API.

## Global Options

| Flag | Env Var | Default | Description |
|------|---------|---------|-------------|
| `--addr` | `VAULT_ADDR` | `http://127.0.0.1:8200` | ZVault server address |
| `--token` | `VAULT_TOKEN` | — | Authentication token |
| `--no-color` | — | `false` | Disable colored output |

## Commands

### System

| Command | Description |
|---------|-------------|
| `zvault status` | Show vault seal status and health |
| `zvault init` | Initialize a new vault with Shamir's Secret Sharing |
| `zvault unseal` | Submit an unseal key share |
| `zvault seal` | Seal the vault (zeroizes all key material) |

### Secrets

| Command | Description |
|---------|-------------|
| `zvault kv put` | Write a secret (key=value pairs) |
| `zvault kv get` | Read a secret by path |
| `zvault kv delete` | Soft-delete a secret |
| `zvault kv list` | List secret keys under a prefix |

### AI Mode (Pro)

| Command | Description |
|---------|-------------|
| `zvault import` | Import secrets from a .env file |
| `zvault run` | Run a command with secrets injected |
| `zvault mcp-server` | Start the MCP server for AI assistants |
| `zvault setup` | Configure an IDE for MCP integration |

### Auth & Tokens

| Command | Description |
|---------|-------------|
| `zvault token create` | Create a child token |
| `zvault token lookup` | Look up current token metadata |
| `zvault approle create-role` | Create an AppRole role |
| `zvault approle login` | Login with role_id + secret_id |

### Encryption

| Command | Description |
|---------|-------------|
| `zvault transit create-key` | Create a named encryption key |
| `zvault transit encrypt` | Encrypt base64-encoded plaintext |
| `zvault transit decrypt` | Decrypt ciphertext |
| `zvault transit rotate-key` | Rotate a key to a new version |

### PKI

| Command | Description |
|---------|-------------|
| `zvault pki generate-root` | Generate a self-signed root CA |
| `zvault pki issue` | Issue a certificate |
| `zvault pki create-role` | Create a PKI role |

### Licensing

| Command | Description |
|---------|-------------|
| `zvault activate` | Activate a Pro/Team/Enterprise license |
| `zvault license` | Show current license status |

## Environment Variables

```bash
export VAULT_ADDR="http://127.0.0.1:8200"
export VAULT_TOKEN="hvs.your-token-here"
```

These can also be passed as flags: `--addr` and `--token`.
